## 1. 개요
- 앱: **발자국(Baljaguk)** / 팀: 어댑독
- 한 줄: **AI 반려동물 동반여행 플래너.** 내 강아지 특성에 맞춰 여행 코스를 설계한다.
- 핵심 컨셉: **"강아지가 주인공"** — 강아지(AI 페르소나)가 화자가 되어 자기소개·코스 제안·일기·연말 결산.
- 차별점: 검색 디렉터리가 아니라 **"펫 제약을 아는" 맞춤 코스 설계.**
- 공모전: 문체부 문화데이터 활용 부문(개발/바이브 코딩). **필수 = 한국관광공사 반려동물 동반여행 API(data.go.kr) 핵심 활용.**

## 5. AI 주입 지점 (5종 모달리티)
1. **에이전트(LLM tool-calling)**: 동선 플래너, 축제 추천
2. **생성형 텍스트(LLM)**: 페르소나(자기소개·일기·결산), 정책 카드 파싱, 오디오 가이드 각색, 케어 알림 문구
3. **생성형 이미지/3D**: 꾸미기(한복), 포토카드, 브이로그, **3D 실사/3D 마스코트 생성**
4. **음성(TTS)**: 강아지 목소리 오디오 가이드
5. **비전 모델**: 견종 인식(이미지·전이학습, 직접 학습), 응급 증상 분석(참고용)

> **안전·규정 판단(입장 판정·폭염·응급 판단)은 AI가 아니라 규칙으로 한다.** AI는 제안·문구만.

## 6. 3D 자산 규칙
- 사진 업로드(A3)·견종 인식(A4)·페르소나(B1/B3)·피규어(F5~F10)·프로필/결산 썸네일은 "업로드한 그 강아지"의 **3D 실사 또는 3D 마스코트**로 표시. 평면 2D 금지.
- 라이브 per-user 생성은 비현실적 → 데모는 체리 사전 제작 에셋(3D 실사 1 + 3D 마스코트 1) 사용. 라이브 생성은 로드맵.
- 생성은 포트 뒤 어댑터(`adapters/outbound/ml/` 또는 외부 3D 생성 API). 실패 시 사전 에셋으로 degrade.

## 8. 아키텍처 & 컨벤션
- 스택: **FastAPI / Python 3.11+**, 헥사고날·클린·DDD, SOLID, Mock-first.
- `domain`은 framework·ML·HTTP를 import 하지 않는다(FastAPI·httpx·pydantic·torch·anthropic 금지).
- 레이어: `adapters/inbound(http)` → `application(use case)` → `domain(model·service·ports)` / `adapters/outbound(kto·llm·ml·weather·vet·mock)` / `config(composition root)`.
- 새 데이터원·모델 추가 = 어댑터 1개 + composition root 1줄. 도메인 무수정(OCP).
- ML/3D: 학습·생성은 `ml/`·외부 API, 추론/호출은 포트 뒤 어댑터. 도메인은 모델을 모름.
- 특징값(활동량·사교성·더위 민감도)은 enum 고정값(UI 직접입력 비활성과 일치). 더위 민감도→RiskPolicy 입력.

## 9. 빌드 순서
1. 도메인(값 객체 → 애그리거트 → RiskPolicy·EntryJudgement·ItineraryComposer) + 포트
2. Mock 어댑터로 데모 핵심 경로 end-to-end 확인
3. 실 API 어댑터(KTO·LLM·ML·날씨·병원·3D) → composition root에서 교체
4. data.go.kr 인증키 먼저 신청. 전주 데모 데이터 미리 정제·캐싱.

## 10. 가드레일 (비협상)
- 안전·규정은 도메인 규칙으로 강제, AI는 제안만.
- **응급/증상 AI는 참고용이며 진단이 아니다.** "반드시 수의사 진료" 항상 표시, 응급 신호(호흡곤란·경련·출혈)는 "즉시 병원" 우선, 특정 약·용량 안내 금지.
- 비밀키·대용량 모델 가중치 커밋 금지(.env, .gitignore).
- Mock 경로·사전 3D 에셋 fallback 유지(데모 생명선).
- 공공데이터 출처 기록(`ml/DATA.md`, UI 출처 태그). 표준데이터에 없는 정보(영업시간 등)는 mock/외부연계 명시.
- 결제·예약은 실제 구현 없이 mock.
