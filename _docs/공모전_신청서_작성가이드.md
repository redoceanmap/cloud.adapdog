### 데이터 전처리·가공·연계 (기획서 5번 "처리 내용")
- **인코딩 정규화**: 펫동반/음식점기본=`utf-8-sig`, 동물병원/배리어프리=`cp949` 혼재 → 인제스트에서 통일.

- **3NF 정규화**: 시설 본체/정책/허용크기 분리, 음식점 본체/이미지(다중값) 분리.

- **명칭 매칭**: 음식점 기본↔이미지 식당명 매칭, 펫동반 식당/카페 카테고리 매칭 → `pet_friendly` 플래그.

| **AI Agent — 동선 플래너** | `route_planner`: 함수호출(`find_pet_friendly_places`) + 멀티턴 대화(목적지→이동수단→숙박→코스), 구조화 JSON 슬롯 추출, 결정형 게이트 안전망 |
| **AI — 정책 카드 파싱** | `policy_card`: 시설 입장규정 자유텍스트 → 표준 배지 코드 LLM 파싱 |
| **규칙 기반(가드레일)** | 입장 판정·응급 증상(`is_diagnostic`=항상 false)은 **AI 아닌 규칙** — 안전·규정 영역 환각 차단 |
| **가공 휴리스틱** | 코리도어 투영, 하버사인 최근접 병원, 식사 리듬(`meal_schedule`), 지역→격자(기상청 nx/ny) |
| **추상화** | 포트(ABC)로 LLM 추상화 → Gemini↔규칙 무중단 교체(장애 시 지속) |

### ★ AI 발전 로드맵 (기획서 4)·6번 — 발전 서사로 기술)
> **톤 주의: "지금은 임시"가 아니라 "완성→고도화" 로 쓴다.**
- **현재(완성·동작):** Gemini 기반 오케스트레이션으로 대화·판정·파싱이 실제 서비스로 완결 동작.
- **향후(고도화):** 서비스가 축적한 **문화데이터(시설 정책 텍스트·입장규정·동선 로그)** 로 학습한 **자체 경량 모델**로 LLM을 점진 대체 → 외부 API 의존·비용 축소, 반려동물 도메인 특화 정확도 향상. → "데이터가 곧 모델 자산"으로 문화데이터 활용 가치를 더 강화.
